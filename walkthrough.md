# Trello-Like Drag and Drop Integration Walkthrough

This document explains how I integrated `@dnd-kit` to provide a robust, Trello-like drag-and-drop experience. `@dnd-kit` is a lightweight, accessible, and highly customizable drag-and-drop toolkit for React.

## 1. The Dependencies
We installed three core packages via `pnpm`:
- `@dnd-kit/core`: Provides the main drag-and-drop context (`DndContext`), sensors (mouse/keyboard tracking), and collision detection algorithms.
- `@dnd-kit/sortable`: Contains hooks (`useSortable`) and components (`SortableContext`) specifically designed for lists where items can be reordered.
- `@dnd-kit/utilities`: Contains helper functions (like `CSS.Transform`) to manage animations and styles while dragging.

## 2. Component Structure

To make drag-and-drop work across multiple lists seamlessly, the architecture consists of three main layers:

### The Board (`components/Board.tsx`)
This is the "Brain" of the operation. 
- **`DndContext`**: Wraps the entire visible area where dragging is allowed. It listens to drag events (`onDragStart`, `onDragOver`, `onDragEnd`).
- **State Management**: It holds the master list of all tasks. When you drag a card over a new column (`handleDragOver`), the Board optimistically updates the React state to move the card instantly without waiting for the server.
- **Server Sync**: When you drop the card (`handleDragEnd`), it calculates the new `order` for all cards and sends a bulk update request (`updateTaskOrders`) to MongoDB so the order persists on refresh.
- **`DragOverlay`**: This renders a "clone" of the card you are currently holding, providing that smooth floating aesthetic.

### The Column (`components/TaskColumn.tsx`)
This represents a single list (e.g., "To Do").
- **`useDroppable`**: Marks this entire `<div>` as a valid "drop zone". Even if the column is empty, you can drop cards into it.
- **`SortableContext`**: Wraps the list of cards. It tells `@dnd-kit`, "These are the items that belong to this column, and they should be sorted vertically (`verticalListSortingStrategy`)."

### The Card (`components/SortableTaskCard.tsx`)
- **`useSortable`**: A special hook that makes an element both draggable *and* a drop target (so it can swap places with other cards).
- We created this wrapper component so we didn't have to clutter your beautiful `TaskCard` UI with complex animation logic. It takes the `transform` and `transition` styles generated by `@dnd-kit` and applies them to a wrapper `<div>`.
- **`touch-none`**: We added this CSS class to prevent mobile devices from trying to scroll the page when you drag a card.

## 3. Database Changes
To maintain the exact order of tasks within a column (just like Trello), we added an `order` field to `models/Task.ts`.
- `order: { type: Number, default: 0 }`
When a task is dragged, the `Board` recalculates the index of every task in the column and updates the database via the `updateTaskOrders` server action.

## 4. Add Another List functionality
Alongside drag-and-drop, we implemented the `AddColumn` component.
- It replaces the static button with a toggleable input field.
- It calls the `addColumn` server action to push a new string to your `User.columns` array in MongoDB.
